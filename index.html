QUIRKI

A memory-safe, data-oriented programming language focused on explicit logic, mutation over return values, and exhaustive pattern matching.

Core Philosophy

  • No classes — everything is plain data
  • Functions never return values; they mutate data
  • Variable names define types
  • Control flow is only pattern matching
  • Enums are tagged unions (no separate union type)
  • Memory-safe, explicit, and exhaustively handled by default

Functions

Functions in Quirki always operate by mutating existing data. There are no return values. 

hero TakeDamage(i_amount)
hero Move(f3_dir)

You can also chain calls vertically:

hero
TakeDamage(i_amount)
Move(f3_dir)

Parameters are readonly by default. Ownership can be taken explicitly:

hero GrabItem(take item)

Variables & Naming

In Quirki, the variable name itself encodes the type. This removes the need for explicit type annotations in most cases. 

// Conventional style
Struct Hero:
   entity : Entity
   weapon : Weapon
   health : Int
// Quirki style
Struct Hero:
   entity
   weapon
   i_health

Enums

Enums are tagged unions and may carry associated data. 

=== <T>? ===
    some(T)
    none
Enum AIState
    Idle
    Patrol(f3_dir)
    Attack(entity)
    Flee(f3_dir, f_speed)
Enum FireWeapon
    success
    no_ammo
    jammed
    overheated

Control Flow

The ? operator is the only control-flow construct in Quirki. It performs exhaustive pattern matching over data. 

? hero.ai_state
| Idle: hero Rest()
| Patrol(dir): hero Move(dir)

Optional values use the same pattern-matching style:

? hero.target
| some(entity): hero Shoot(entity)
| none: hero Search()

Anonymous Match Values

The # symbol introduces a temporary, anonymous value for matching. This is useful when you don’t want to name an intermediate result. 

?: # GetTotalAmmo(hero)
| # > 0: hero Shoot()
| else: hero Reload()

Booleans are enums with syntactic sugar: 

? has_ammo CheckAmmo(hero)
    hero Shoot()
| else:
    hero Reload()

Errors

There are no exceptions or generic Result<T, E> types. Errors are expressed as domain-specific enums. 

=== LoadAsset ===
    success(asset)
    error(string)
? result LoadTexture(path)
| success(asset): loadedAssets(take asset)
| error(msg): log Write(msg)
